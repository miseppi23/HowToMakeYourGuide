1 Introducción

Los informes HTML son cada vez más populares ya que permiten una apariencia profesional rápida, incluir gráficos y mapas interactivos. Dashboard Power BI se usa comúnmente dentro de SRK (Conceptos básicos del Dashboard), es muy útil especialmente para monitorear proyectos.

Pero los informes HTML tienen varias ventajas:

  • accesibles desde cualquier máquina (computadora, tabletas, teléfono)
  • actualízalo sobre la marcha, así te aseguras de que el cliente siempre esté usando la última versión
  • puedes incluir fácilmente una opción para descargar tu informe como un documento pdf, word o powerpoint ya que los clientes suelen usar nuestras figuras para sus propios informes internos

2 Configura tu entorno

2.1 Instalar Quarto

Quarto es un sistema de publicación científica y técnica de código abierto diseñado para crear documentos, informes, presentaciones y sitios web de alta calidad. Se integra perfectamente con herramientas y lenguajes como R, Python y Jupyter, lo que permite a los autores combinar texto, código y visualizaciones en un solo documento.

  1. Instala Quarto en tu máquina. Ve a aquí. Tendrás que reiniciar tu máquina.
Note

Documentación extensa está disponible en línea.

2.2 Instalar extensiones

Para usar Quarto con VS Code, necesitaremos instalar la extensión.

  1. Ve a la barra lateral en VS Code y selecciona Extensiones o CTRL+X.
  2. Busca Quarto e instala la extensión.

3 Configura tu sitio web

3.1 Configura tu página principal

  1. Ve a la ventana Explorer, haz clic derecho y selecciona New Folder

  1. El nombre de tu carpeta será el nombre de tu repositorio en GitHub (ver Curso StartCodingToday). En este caso, lo nombraré HowToMakeYourGuide.

  2. Para comenzar a crear tu página web, crea un nuevo archivo .qmd. Haz clic derecho en tu carpeta recién creada y selecciona New file. Aquí lo llamaré HowToMakeYourGuide_Span.qmd.

  1. Luego necesitaremos crear e inicializar un archivo .yml que contendrá la información del sitio web: el título, los paneles de la barra de navegación y las páginas web (.qmd) que configuraremos a continuación. Crea un nuevo archivo y nómbralo **_quarto.yml**

  1. Escribe lo siguiente en el archivo **_quarto.yml**
project:
  type: website
  output-dir: docs

website:
  title: "Cómo Hacer Tu Guía"
  navbar:
    left:
      - href: HowToMakeYourGuide_Span.qmd
        text: Span

format:
  html:
    theme: cosmo
    toc: true
Important

Las líneas output-dir: docs significan que los archivos que Quarto generará para nuestro sitio web se guardarán en una carpeta llamada docs. Asegúrate de que esta línea esté presente en tu archivo .yml ya que será clave para publicar tu sitio en GitHub Pages.

  1. Abre tu archivo .qmd HowToMakeYourGuide_Span.qmd. Y usa el atajo de teclado CTRL+SHIFT+K. Este atajo debería haberse instalado junto con la extensión de Quarto y se abrirá una ventana lateral con una vista previa de tu sitio web. Esto facilitará la construcción.

Note

El archivo .yml se puede actualizar en cualquier momento. Si deseas agregar una página, cambiar tu título, formato, etc. Lo gestionaremos a lo largo de la creación del sitio web.

3.2 Agregar otra página

Si deseas crear otra página que estará disponible en la barra de navegación en la parte superior de tu sitio web.

  1. Crea un Nuevo archivo y nómbralo de acuerdo con el contenido de tu segunda página. En este caso: HowToMakeYourGuide_Eng.qmd

  2. Luego necesitarás vincularlo en tu archivo .yml.

project:
  type: website
  output-dir: docs

website:
  title: "Cómo Hacer Tu Guía"
  navbar:
    left:
      - href: HowToMakeYourGuide_Span.qmd
        text: Span
      - href: HowToMakeYourGuide_Eng.qmd
        text: Eng

format:
  html:
    theme: cosmo
    toc: true
  1. CTRL+SHIFT+K. Ahora puedes ver tus dos páginas en la parte superior.

4 Construye tu página web

4.1 Sección de encabezado

Ahora podemos construir nuestra página web.

  1. En el archivo .qmd, primero especificaremos el título y el autor.
---
title: "Cómo Hacer Tu Guía"
author:
  - name: Marion Iseppi, Consultora Senior 
---
Note

La sección de encabezado en el archivo .qmd está separada por —. Asegúrate de que estén presentes y que se respeten las sangrías.

  1. CTRL+SHIFT+K debería actualizar tu vista previa

  1. Puedes agregar tantos detalles como necesites: subtítulo, fecha, especificar más sobre el autor como grado, afiliación, etc.

4.3.3 Notas

Puedes agregar notas a lo largo de tu sitio para una mejor comprensión usando callout.

  1. Escribe callout en tu archivo. Con la extensión de Quarto, te propondrá una lista de posibilidades. Intenta y selecciona callout-note.

  1. Escribe tu nota dentro de los :::
::: {.callout-note}
Documentación extensa está disponible [en línea](https://quarto.org/docs/guide/). 
:::
  1. Presiona CTRL+SHIFT+K.

4.3.7 Mostrar bloque de código delimitado

Es posible que desees incluir bloques de celdas sin ejecutarlos. En este caso, agrega un . antes de python.

Y se renderizará de la siguiente manera:

import plotly.graph_objects as go
import numpy as np

# Generar datos de muestra
np.random.seed(42)
data = np.random.randn(500)

# Crear un histograma
fig = go.Figure()

# Agregar traza para el histograma
fig.add_trace(go.Histogram(
    x=data,
    name='Datos de Muestra',
    marker=dict(color='hotpink', line=dict(width=0.5))
))

# Personalizar diseño
fig.update_layout(
    title='Histograma usando Plotly',
    xaxis_title='Valor',
    yaxis_title='Frecuencia',
    template='plotly_white'
)

# Mostrar el gráfico
fig.show()

6 Publicar en GitHub

6.1 Prepara tu proyecto para GitHub

Ahora que has creado tus páginas y que tu repositorio ha sido publicado en GitHub, desplegaremos tu sitio web usando GitHub Pages.

  1. En tu terminal ingresa:
quarto render

Puedes ver que ha creado una carpeta docs en la raíz de tu repositorio.

Note

Si aún estás visualizando la vista previa de tu sitio web usando CTRL+SHIFT+K, notarás que tu terminal está ocupada. Para detener el comando, haz clic en tu terminal y usa CTRL+C.

  1. Ve a la pestaña CONTROL DE FUENTES en la barra lateral izquierda de VS Code. Verás que hay varios cambios almacenados. Los vas a confirmar en GitHub.

  1. Haz clic en el signo + Stage All Changes y COMMIT. Se abrirá una ventana COMMIT_EDITMSG. En la primera línea, ingresa una breve descripción de los cambios. Luego guarda y cierra la ventana.

  1. Luego haz clic en Sync Changes. Una ventana te advertirá que VS Code va a enviar los cambios a GitHub, haz clic en OK.

6.2 GitHub Pages

  1. Ve a tu repositorio en línea y haz clic en el recién publicado. Luego en la barra superior ve a Settings.

  1. En Settings, haz clic en Pages que se encuentra en la barra de navegación izquierda.

  1. En Pages, especifica la rama a main y la carpeta a /docs. Luego haz clic en Save. Deberías ver aparecer una banda azul GitHub Pages saved.

  1. Actualiza tu página, ahora deberías ver un enlace a tu sitio web construido como sigue: https://tu-usuario.github.io/tu-nombre-del-repositorio. Ve a tu sitio web.
